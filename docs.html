Request flow

The request flow is the merchant-side simulation. It creates a payment context before anything is paid.

1. Enter the request

Provide the amount, a reason, and the payer or business name. The flow uses that data to build a readable request.

2. Set the rules

Choose an expiry and optional settings such as partial payment. These settings shape the simulated request summary.

3. Share the result

The request generates a shareable code. Users can copy it, send it by message, or show it on screen.

Receive flow

The receive flow shows what happens after a request is created. It groups the wallet, methods, and payment history together.

Wallet view Balance, pending items, and settled payments.
Receive methods Payment code, link, QR, and bank details in one place.
